## 먼저 결론부터

> **구매조직과 플랜트는 같은 레벨이 아니다.**
> 둘은 **목적 자체가 다른 별개의 조직 단위**다.

```
플랜트      →  "어디서 재고를 관리하고 생산하냐"
구매조직    →  "누가 공급업체와 계약하고 발주하냐"
```

---

## 1. 플랜트 vs 구매조직 — 삼성전자 예시

### 플랜트 = 공장/사업장 ("어디서")

```
삼성전자
├── 수원 공장    (Plant 1000)  ← 스마트폰 생산
├── 구미 공장    (Plant 2000)  ← TV 생산
└── 평택 공장    (Plant 3000)  ← 반도체 생산
```

- 각 공장은 **독립적으로 재고를 관리**
- 수원 공장 재고 ≠ 구미 공장 재고 (Plant가 다르면 재고도 분리)
- MRP(자재소요계획)도 Plant 단위로 실행

---

### 구매조직 = 구매 협상 권한을 가진 팀 ("누가")

```
삼성전자
└── 중앙구매팀 (Purchasing Org P100)
    "공급업체 A에 연간 1조 구매할테니 단가 낮춰줘" 협상하는 팀
```

- 공급업체와 **가격, 납기, 조건을 협상하고 계약**하는 단위
- 구매오더(PO) 발행 권한 보유
- 인포레코드, 계약서도 구매조직 단위로 관리

---

## 2. 중앙구매 vs 분산구매

### ✅ 중앙구매 = 구매조직 1개 → 여러 플랜트 담당

```
구매조직 P100 (삼성 중앙구매팀)
   ├── Plant 1000 (수원 공장)  발주 가능 ✅
   ├── Plant 2000 (구미 공장)  발주 가능 ✅
   └── Plant 3000 (평택 공장)  발주 가능 ✅
```

> ⚠️ "구매조직 1개 = 플랜트 1개"가 중앙구매가 **아니다!**
> **구매조직 1개가 여러 플랜트를 커버**하는 것이 중앙구매다.

---

### ✅ 분산구매 = 플랜트마다 각자의 구매조직이 있음

```
구매조직 P100 (수원 구매팀)  →  Plant 1000만 담당
구매조직 P200 (구미 구매팀)  →  Plant 2000만 담당
구매조직 P300 (평택 구매팀)  →  Plant 3000만 담당
```

- 각 공장이 독립적으로 공급업체와 협상하고 발주

---

## 3. 중앙구매 vs 분산구매 비교

| 항목 | 중앙구매 | 분산구매 |
|------|----------|----------|
| 공급업체 협상 | 본사가 통합 협상 | 각 공장이 개별 협상 |
| 구매 단가 | 대량 구매로 유리 | 소량이라 불리할 수 있음 |
| 현장 대응 속도 | 느림 (본사 승인 필요) | 빠름 (현장에서 즉시 결정) |
| 프로세스 관리 | 단순 (구매조직 1개) | 복잡 (구매조직 다수) |
| 공급업체 마스터 | 통합 관리 | 중복 관리 부담 |
| 적합한 상황 | 표준화된 대량 구매 | 현지 대응이 중요한 경우 |

---

## 4. 구매조직 유형 정리

| 유형 | 설명 | 예시 |
|------|------|------|
| 중앙구매 | 1개 구매조직 → 여러 Company Code의 Plant 담당 | 그룹사 통합 구매 |
| 회사코드별 구매 | 1개 구매조직 → 1개 Company Code 내 여러 Plant | 법인별 구매팀 |
| 플랜트별 분산구매 | 1개 구매조직 → 1개 Plant만 담당 | 공장별 독립 구매 |
| 참조구매조직 | 단가 마스터만 관리, 실제 발주는 하위 조직 수행 | 본사 글로벌 계약 + 지역 발주 |

---

## 5. SAP에서 구매조직 관련 T-Code

| T-Code | 설명 |
|--------|------|
| OX08 | 구매조직 생성/변경 |
| OX01 | 구매조직 → Company Code 할당 |
| OX17 | 구매조직 → Plant 할당 |
| OME4 | 구매그룹 생성/변경 |

---

## 6. 핵심 요약

```
플랜트       =  재고/생산 단위  (공장, 창고)
구매조직     =  계약/발주 권한 단위  (구매팀)

중앙구매     =  구매조직 1개  →  여러 플랜트 커버
분산구매     =  구매조직 여러 개  →  각자 플랜트 1개씩 담당
```

> 컨설턴트 관점에서 구매조직 설계 시 핵심 질문:
> **"구매 협상력을 어디에 집중할 것인가?"**
> 구매조직이 많아질수록 공급업체 마스터 관리 부담이 급증하므로
> 가능한 한 구매조직 수를 최소화하는 것이 모범 사례다.
