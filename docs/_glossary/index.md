# SAP MM 용어 사전 (Glossary)

알파벳 / 가나다 순 정렬. Full Name(영문 풀네임)과 한국어 설명을 함께 기재합니다.

---

## A

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **AP** | Accounts Payable | 매입채무. 공급업체에 지급해야 할 채무 계정 |

---

## B

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **BOM** | Bill of Materials | 자재명세서. 완제품 또는 반제품을 구성하는 자재 목록과 수량을 정의 |
| **BP** | Business Partner | 비즈니스 파트너. 공급업체, 고객 등 거래처를 통합 관리하는 SAP S/4HANA 방식 |
| **BSX** | (Inventory Posting Key) | 재고 계정 키. GR 전기 시 재고자산 계정(차변)을 결정하는 계정 결정 키 |

---

## C

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **Client** | Client (클라이언트) | SAP 시스템의 최상위 조직 단위. 모든 데이터는 Client 안에 존재 |
| **Company Code** | Company Code (회사 코드) | 독립적인 회계 장부를 보유하는 법인 단위. FI의 기본 조직 단위 |
| **Consignment** | Consignment (위탁) | 공급업체 재고를 자사 창고에 보관하다가 출고(사용) 시점에 대금을 정산하는 조달 방식 |

---

## D

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **DepReq** | Dependent Requirement | 종속 소요량. BOM 전개 시 상위 자재 수요로부터 하위 구성 자재에 자동 생성되는 수요 |

---

## E

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **ERS** | Evaluated Receipt Settlement | 자동 송장 정산. GR 수량과 PO 단가를 기준으로 송장을 자동 생성·정산하는 방식 |
| **ESOKZ** | (Info Category Key) | Info Record 유형 코드. 0=Standard, 1=Pipeline, 2=Consignment, 3=Subcontracting |

---

## F

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **FI** | Financial Accounting | 재무회계 모듈. GR·IV 전기 시 회계 전표 자동 생성 |
| **Fixed Source** | Fixed Source (고정 공급원) | Source List에서 MRP 실행 시 우선 선택되도록 지정된 공급업체 |

---

## G

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **GR** | Goods Receipt | 입고. 공급업체로부터 자재를 수령하여 재고로 등록하는 행위. T-code: MIGO (Movement 101) |
| **GR/IR** | Goods Receipt / Invoice Receipt | 입고/송장 정산 계정. GR 후 IV 전까지의 미결제 잔액을 관리하는 중간 정산 계정 |
| **GR-Based IV** | GR-Based Invoice Verification | 입고 기준 송장 검증. GR 수량 범위 내에서만 IV 가능하도록 통제하는 설정 |

---

## I

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **Info Record** | Purchasing Info Record (구매 정보 레코드) | 자재-공급업체 조합의 단가, 납기, 허용 오차 등 구매 조건을 저장하는 기준 정보. T-code: ME11 |
| **Incoterms** | International Commercial Terms | 국제 무역 거래 조건. 운송 책임과 비용 부담 범위를 정의 (예: FOB, CIF, DDP) |
| **IV** | Invoice Verification | 송장 검증. 공급업체 청구서를 PO·GR과 비교하여 검증하고 채무를 계상. T-code: MIRO |

---

## L

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **Lot Size** | Lot Size (로트 크기) | MRP가 계획 오더/PR 수량을 결정하는 방식. EX(순소요량), FX(고정수량), HB(최대재고 보충) 등 |

---

## M

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **Material Master** | Material Master (자재 기준 정보) | SAP 내 자재에 관한 모든 정보를 관리하는 중앙 기준 정보. 약 37개 View, 500여 필드 구성 |
| **MDM** | Master Data Management | 기준 정보 관리. 정확성·신속성·신뢰성·활용성을 갖춘 마스터 데이터를 운영하는 체계 |
| **Movement Type** | Movement Type (이동 유형) | 자재 이동의 성격을 정의하는 코드. 101=PO 기준 입고, 261=생산 오더 출고, 551=폐기 등 |
| **MRP** | Material Requirements Planning | 자재 소요량 계획. 수요와 재고를 분석하여 구매 요청(PR) 또는 계획 오더를 자동 생성 |
| **MRP Controller** | MRP Controller (MRP 콘트롤러) | 자재 마스터 MRP 1 View에 지정하는 MRP 담당자. MRP 결과 예외 메시지 처리 책임자 |
| **MRP Type** | MRP Type (MRP 유형) | MRP 계획 방식 코드. PD=수요 기반, VB=재발주점, VM=자동 재발주점, ND=미적용 |

---

## N

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **Net Requirement** | Net Requirement (순소요량) | 총소요량에서 현재고와 공급 예정량을 차감한 실제 조달 필요 수량 |

---

## P

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **P2P** | Purchase to Pay | 구매부터 지급까지의 전체 구매 프로세스 흐름. PR → RFQ → PO → GR → IV → 지급 |
| **PDT** | Planned Delivery Time (계획 납기 시간) | 발주 후 입고까지 소요되는 일수. MRP 납기 역산의 핵심 파라미터. Info Record 값이 우선 |
| **PIR** | Planned Independent Requirement | 계획 독립 소요량. MRP에서 사용하는 예측 수요. T-code: MD61에서 수동 입력 |
| **Pipeline** | Pipeline (파이프라인) | 전기·수도·가스처럼 파이프를 통해 연속 공급되는 자재의 조달 유형 |
| **Plant** | Plant (플랜트) | 자재가 생산·보관·조달되는 물리적 단위. 구매 조직과 N:M 관계 |
| **PlOrd** | Planned Order (계획 오더) | MRP가 내부 생산 또는 외부 조달을 위해 자동 생성하는 계획 문서 |
| **PO** | Purchase Order | 구매 발주서. 공급업체와의 공식 구매 계약 문서. T-code: ME21N |
| **PR** | Purchase Requisition | 구매 요청서. 내부 부서가 구매팀에 자재 조달을 요청하는 문서. T-code: ME51N |
| **PReq** | Planned Purchase Requisition | 계획 구매요청. MRP가 외부 조달 필요 시 자동 생성하는 PR |
| **Purchasing Group** | Purchasing Group (구매 그룹) | 구매 업무를 담당하는 실무자 또는 부서. PR/PO에 기본값으로 자동 입력 |
| **Purchasing Org** | Purchasing Organization (구매 조직) | 공급업체와 구매 조건을 협상하는 조직 단위. Info Record, 계약의 기준 단위 |

## R

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **Release Strategy** | Release Strategy (승인 전략 / 릴리스 전략) | PR·PO 금액 또는 조직 기준으로 결재 단계를 정의하는 SPRO 설정 |
| **Reorder Point** | Reorder Point (재발주점) | VB(재발주점 계획) 방식에서 재고가 이 수량 이하로 떨어지면 자동으로 PR이 생성되는 기준 수량 |
| **RFQ** | Request for Quotation | 견적 요청서. 공급업체에 가격·납기 조건을 요청하는 문서. T-code: ME41 |

## S

| 약어 / 용어 | Full Name | 설명 |
|-----------|----------|------|
| **Safety Stock** | Safety Stock (안전 재고) | 리드타임 변동 및 수요 불확실성에 대비한 최소 보유 재고량. MRP 2 View에서 설정 |
| **SchedA** | Scheduling Agreement (스케줄링 합의) | 납품 일정이 포함된 장기 구매 계약. MRP 공급 요소로 활용 |
| **Source List** | Source List (소스 리스트 / 공급원 목록) | 자재별 허가된 공급업체 목록. Fixed/Blocked 설정으로 MRP 공급원 통제 가능. T-code: ME01 |
| **SPRO** | SAP Project Reference Object | SAP 커스터마이징 메뉴 경로. 시스템 설정(Configuration) 작업의 진입점 |
| **STO** | Stock Transfer Order | 재고 이동 오더. 플랜트 간 또는 회사 코드 간 자재 이동 시 사용 |
| **Subcontracting** | Subcontracting (외주) | 원자재를 공급업체에 사급하여 가공 후 반제품/완제품으로 수령하는 조달 방식 |

## MRP 유형 코드 요약

| 코드 | 유형 | 설명 |
|------|------|------|
| PD | MRP | 수요 기반 계획. 종속/독립 소요량에 따라 PR/계획 오더 자동 생성. 가장 일반적 |
| VB | Reorder Point Planning | 재발주점 계획. 재고가 재발주점 이하로 떨어지면 보충 PR 자동 생성 |
| VM | Automatic Reorder Point | 자동 재발주점. 과거 소비량 기반으로 재발주점을 자동 계산 |
| ND | No Planning | MRP 미적용. MRP 실행 대상에서 제외 |

## Info Record 유형 (ESOKZ)

| 코드 | 유형 | 설명 |
|------|------|------|
| 0 | Standard | 일반 외부 구매 오더용 |
| 1 | Pipeline | 파이프라인 공급 자재 (전기, 수도, 가스) |
| 2 | Consignment | 위탁 재고 - 출고 시점에 단가 정산 |
| 3 | Subcontracting | 외주 가공 - 사급 자재 단가 관리 |
